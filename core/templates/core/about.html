About Teal CRM

Teal CRM is a compact, educational Django app demonstrating multi-tenant patterns with teams, plan-based limits, and simple lead-to-client workflows — built to be clear, extendable and easy to run locally.

Core ideas

  • Teams (workspaces) — isolate users' data and apply plan limits per team.
  • Plans — control how many leads/clients a team can create; useful for billing demos.
  • Lead → Client — a simple conversion flow to promote qualified leads into clients.

For users

Sign up to get a personal team and start adding leads. Use the dashboard to track priorities and statuses, convert leads into clients, and manage teammates if you need collaboration.

Architecture & conventions

The app follows a simple Django layout: apps for leads, clients, teams, dashboard and user profiles. Views use function-based views and templates are styled with Tailwind (CDN). Environment configuration is handled via python-decouple.

Important domain rules:

  • All user data is scoped by team where possible — teams act as the tenant boundary.
  • Plan limits are enforced per team for leads and clients.

Tech stack

Python • Django • PostgreSQL • Tailwind (CDN)

Getting started

Clone, create a virtualenv, install requirements and run migrations — see README for details.

Admin

Use Django admin to create Plans and seed Teams for quick testing.

Contributing

  1. Fork the repository and create a feature branch.
  2. Add tests where appropriate and run the test suite locally.
  3. Open a pull request with a clear summary and motivation for the change.

This project is intended for learning and demos. For production use you should harden authentication, add proper billing, tenancy isolation and comprehensive tests.
